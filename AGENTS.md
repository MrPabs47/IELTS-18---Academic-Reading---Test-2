Project rules:
1. Preserve the existing standalone HTML architecture and styling unless explicitly asked to change them.
2. Reuse existing question components whenever possible.
3. Make the smallest possible code changes.
4. Do not introduce external libraries or dependencies.
5. Keep everything self contained in one HTML file unless explicitly asked otherwise.
6. Preserve current scoring, review, navigation, and answer checking behaviour.
7. For new question types, build the smallest reusable component possible.
8. Do not rewrite working sections unnecessarily.
9. If a requirement is unclear, prefer a minimal safe implementation.
10. When editing Cambridge 18 Academic Reading Test 2, work section by section.
